# Anna.i Brand Identity & Design Language Guidelines (BrandID)

## 1. Core Brand Essence & Positioning
* **Brand Name:** Anna.i [cite: 1, 2]
* **Tagline / Core Proposition:** The Operating System for the Modern Household [cite: 2]
* **Mission & Vision:** Redefining the domestic economy through a category-defining platform play, transforming how families live in the AI era so they can be present, not just efficient [cite: 2].
* **Core Mechanisms:**
  1. **Memory:** The Household Graph (storing preferences, vendor history, preference mapping, and routine recognition) [cite: 2].
  2. **Closed-Loop:** The Trust Engine (ensuring accountability via task dispatch, verification through mandatory photo proof, and escrow) [cite: 2].
  3. **Predictive:** The AI Layer (evolving from reactive approval to anticipatory routing and decision automation) [cite: 2].
* **Brand Voice & Personality:** Rigorous, direct, highly structured, technically sophisticated, yet deeply human and anchored in trust, reliability, and precision [cite: 2].

## 2. Color Palette & Typography

### 2.1 Color Palette
The color language reflects a premium, modern, and desaturated natural aesthetic, avoiding neon or overly saturated primary colors [cite: 1].

| Tier | Color Name | Hex Code | Visual Application & Hierarchy [cite: 1] |
| :--- | :--- | :--- | :--- |
| **Primary** | Sage Green / Moss | `#A8BBAE` | Main branding element, primary interactive components, active states, and focus elements. |
| **Secondary** | Deep Slate / Charcoal | `#203132` | Dark text contrast, core structural elements, background headers, and high-impact containers. |
| **Tertiary** | Off-White / Alabaster | `#F9F8F3` | Surface areas, backgrounds, card components, and subtle section dividers. |
| **Neutral** | Absolute White | `#FFFFFF` | Canvas backgrounds, badge structures, and text labels over dark fills. |

### 2.2 Typography Hierarchy
The brand uses a combination of modern geometric sans-serif typefaces for user-facing surfaces and a highly structured monospace typeface for labels and structured technical feedback [cite: 1].

* **Headline Typeface:** **Manrope** [cite: 1]
  * *Characteristics:* Geometric, clean, open-aperture, high readability.
  * *Usage:* Screen titles, h1, h2 sections, and major numerical metric declarations.
* **Body Typeface:** **Manrope** [cite: 1]
  * *Characteristics:* Balanced metrics, legible at small scales on mobile and web viewports.
  * *Usage:* Descriptive paragraphs, system logs, explanatory text block copies.
* **Label & Data Typeface:** **JetBrains Mono** [cite: 1]
  * *Characteristics:* Monospace, clear character differentiation, distinct geometric technicality.
  * *Usage:* Secondary system metrics, status labels, token identifiers, tracking data, and technical AI logs.

## 3. UI Component Styles & States

### 3.1 Button Variants & Controls [cite: 1]
* **Primary Button:** Filled block with the brand's primary color (`#A8BBAE`) or solid secondary dark slate (`#203132`) depending on context. Text is centered and bold.
* **Secondary Button:** Muted or lower contrast fill option used for secondary choices.
* **Inverted Button:** White or light text/border treatment applied on deep backgrounds (`#203132`).
* **Outlined Button:** Transparent background with a crisp border matching the text or icon color (`#203132` or `#A8BBAE`). Used for secondary dashboard actions or optional paths.
* **Search / Inputs:** Fully rounded pill-shaped or softly rounded rectangular inputs featuring a minimalist search icon ($Q$) and placeholder copy.

### 3.2 Visual Containers & Layout Geometry
* **Corner Radii:** Soft, rounded rectangular borders across cards, screens, and dialog prompts (typically between `12px` to `24px` radius).
* **Zebra Striping & Borders:** Thin, light-grey or low-contrast desaturated borders outlining key data points to split high-density analytical dashboards.
* **Layout Structure:** Linear, clean whitespace separation. Heavy website UI patterns (like massive hero sections or distracting gradients) are stripped away to focus on utility, data density, and operational speed.

## 4. Iconography & Freehand Style Elements

### 4.1 UI Navigation Icons [cite: 1]
Dashboard navigation relies on clean, single-line stroke geometric iconography:
* **Home / Dashboard:** A clean, simplified pentagonal house silhouette containing an inner horizontal line or nested structure.
* **Search / Discovery:** A perfectly circular loupe with an angled sharp handle ($Q$).
* **Identity / Profile:** An abstract human shoulder-and-head profile silhouette.
* **System Operations / Tools:** Intersecting nodes, modular square arrangements, trash cans, edit pencils, and quick action floating action buttons (FABs).

### 4.2 Freehand & System Overlay Elements [cite: 1]
* **Aesthetic Context:** The user interface combines precise technical layouts with loose, hand-drawn or clear single-stroke icon assets to establish a casual yet sharp aesthetic.
* **Application:** Used inside personalized onboarding flows and service classification categories (e.g., stylized line icons representing *Cleaning & Laundry*, *Aircon & Maintenance*, *Caregiving*).
* **Execution:** Lines maintain consistent weight, minimal intersecting visual noise, and scale fluidly across dashboards to indicate quick system tips, human-centric focus points, and user actions.

## 5. Strategic Context & Target Application

These design guidelines must be systematically applied across all three key platform interfaces [cite: 2]:
1. **User Onboarding & Mobile App:** Clean, high-whitespace step-by-step wizard paths leveraging warm backgrounds (`#F9F8F3`), distinct rounded options, and predictive status animations [cite: 2].
2. **Vendor SaaS Portal:** High-efficiency, structured operational views optimizing vendor utilization, scheduling, automated billing, and instant payouts using structural monospace font hierarchies [cite: 2].
3. **Operations Control Center:** Ultra-dense live analytical dashboards tracking infrastructure health, AI Learning Indices, dispatch success rates, and real-time validation proofs [cite: 2].
